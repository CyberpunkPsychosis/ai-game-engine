# AI 素材工具清单(侧重"按参考图出同风格")

> 离线整理,无法核对最新价格/链接;名字与定位准确,自行搜索。
> 核心诉求:**给 AI 看我找的素材 → 生成风格差不多的新素材**。下面按这个能力标注。

## 先懂 3 种"风格匹配"原理(决定像不像)
1. **风格/图像参考(IP-Adapter / Style Reference / Image Prompt)**
   - 喂 1~几张参考图,直接出相似风格。**免训练、快**,像度 ~70–85%。
   - 适合:快速试、风格不要求 100% 锁死。
2. **训练专属模型(LoRA / Custom Model / Fine-tune)** ⭐ 最稳
   - 用**一组**参考图(20–50 张)训一个"你的风格/角色"模型,以后每次生成都贴合。像度 ~85–95%。
   - 适合:成套素材、需要长期一致(主角多动作、统一世界观)。
3. **ControlNet / img2img**:控制**结构/姿势/线稿/轮廓**(配合上面两种用),让新图"形"也对。

> 一致性公式:**训练模型(锁风格) + ControlNet(锁姿势/结构) + 固定底模/种子**。

## 自己跑(要 GPU 或云 GPU,控制力最强、免费)
- **Stable Diffusion / Flux + ComfyUI**:训 **LoRA**(锁你的风格)+ **IP-Adapter**(参考图)+ **ControlNet**(姿势)。
  - 训 LoRA 工具:**Kohya_ss**;模型社区:**Civitai**(也能在线训/生成)。
  - Flux 的 **Redux**(图像变体/风格)适合"照参考出相似"。
- 这条最强但要折腾;你没 GPU 就用上面的云端版(Scenario 本质就是把这套管线托管了)。

## 按你的场景怎么选
- **想最省心 + 风格最稳(成套素材/主角多动作)** → **Scenario**(训模型)或 **Recraft**(从图建风格)。
- **快速试、不想训** → 任意工具的**风格参考/IP-Adapter**;或 **Midjourney `--sref`/`--cref`**。
- **走像素风 + 要动画帧** → **PixelLab**。
- **要可控的姿势/连招关键帧** → 上面任一 + **ControlNet(OpenPose)**。

## 重要提醒(踩坑经验)
- **一致性 > 帧数 > 画质**:先把"是同一个角色/同一画风"锁住,再谈细节。
- **角色动画别纯生成**:出**几张关键姿势**(idle/跑/攻击)→ 我这边**绑骨/补间**成动画,比逐帧生成稳。
- **格式**:透明底、侧视朝右、统一画幅、脚底对齐、一张图一个姿势(见 `CHARACTER_POSES.md`)。
- 出图/出视频后,你用 `tools/sprite-keyer` 选帧+抠图+对齐,导透明横条发我,我切帧接进引擎(见 `SPRITE_PIPELINE.md`)。
- 商用要留意各工具的**授权条款**(尤其用别人素材训模型属衍生)。

## ⚠️ 实测结论(2026-06,fal.ai 跑了一整轮,真金白银验过)

> 这次想用 AI 给「刹那」主角(狐面剑士)出一套从 idle 起步的动作素材,在 fal.ai 上把当时主流的图像/视频/精灵模型几乎全试了一遍。
> **一句话结论:单张精灵 AI 能做;让单张图『动起来且帧间一致』,现阶段 AI 还做不到能直接用的程度。** 所以美术继续走「现有素材 / 自己画」,AI 动画暂不投入。

### 试过哪些、结果如何
| 路线 | 用到的模型(fal) | 结果 | 为什么不可行 |
|---|---|---|---|
| **立绘→佩刀/改姿势(图像编辑)** | FLUX Kontext、Seedream 4.5、Qwen-2511、Gemini 3.1 Flash、**Nano-banana-pro**、Flux-2 | **可用** ✅ | 单张质量够当游戏素材。Nano-banana-pro 保结构最稳、Seedream 4.5 画质最高但爱自由重摆姿势(常拔刀/转正面),需反复钉提示词 |
| **纯图像出 idle 序列帧** | Nano-banana-pro(一次出多帧横条) | 单帧一致性惊人,**但不能当动画** ❌ | 每帧各自重画→扣环/衣褶/轮廓逐帧飘移(对齐后残差仍 ~60,肉眼明显抖),不是平滑呼吸 |
| **通用视频模型做 idle 循环** | Seedance 1.5 Pro(首尾帧=同图+camera_fixed)、LTX-2 蒸馏/常规 | 没跑出可用成品 ❌ | Seedance 能出但动作不可控、按条收费($0.26/条)迭代贵;**LTX 开源端点拥堵**(免费池排队 38 位、付费队列也排 20+),不适合反复试 |
| **专用精灵动画工具(骨骼/绑定)** | Ludo.ai(我们有 MCP)、PixelLab | 理论最对口、未跑通 ⚠️ | Ludo 的 MCP **没配 API key**;按 credit 计费偏贵,PixelLab 同样要另注册付费。骨骼方案能解一致性,但要为这条单独花钱 |

### 为什么"动画"是死结(核心瓶颈)
- **逐帧生成 = 逐帧重画**:图像模型没有"同一对象在时间上连续"的概念,每帧都重新画一遍角色,细节必然漂移。这是原理性的,提示词/换模型都治不好。
- **通用视频模型**:能保时间连续,但①动作不可控(给你随机演)②像素/侧视/锁机位等游戏约束压不住③便宜的开源版(LTX)排队严重、可用的(Seedance/Kling)按条收费,迭代成本高。
- **真正能解一致性的是骨骼/绑定**(Ludo/PixelLab/自己绑骨),但这要么花钱要么要人工,已超出"AI 一键出"的范畴。

### 现阶段的现实选择(就这两条)
1. **用现有素材**:成品精灵表(itch.io / 各素材站),风格统一、自带多动作帧,直接切帧入库(见 `ASSET_SOURCING.md` / `SPRITE_PIPELINE.md`)。
2. **自己画**:用户在自学美术。AI 出**几张关键姿势静帧**当参考没问题,**补间/动起来这步还得人来**(或将来 AI 动画一致性过关再说)。
- 主角当前仍**色块占位**,先把「凝固之物即地形」核心玩法打磨好玩,美术不挡路。

### 留给将来的人(什么时候值得回头再试)
- 出现**便宜、可控、帧间一致**的精灵动画专用模型/服务(或 Ludo/PixelLab 降价、白嫖额度够用)时,再回来跑。
- 那之前,**别再在"通用视频/图像模型逐帧出角色动画"上烧钱**——这条已验证此路不通。
- 本轮 fal 充值 $10、实际花约 $2.5,key 仍可用;单图编辑(Nano-banana-pro / Seedream 4.5)这块是**真能用**的,需要某张静态立绘/姿势图时可单独用,别扩到动画。
