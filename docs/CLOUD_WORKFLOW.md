# 云端出素材工作流(主推 Scenario,含备选)

> 目标:**云端按"参考图→同风格"生成素材 → 传到 `incoming/` → 引擎侧入库接入**。
> 本文可当人工指南，也可当**本地 Claude 的执行说明**(本地有网,能调 API / 开网页)。
>
> ⚠️ 说明:本仓库的云端 Claude 离线,无法核对各服务最新 API/界面。**API 的具体端点/参数,以各家官方文档为准**(下面标了"查官方文档"的地方)。

---

## 0. 分工
- **本地(你 + 你电脑的 Claude)**:在云服务上生成素材(出图)。
- **产出落地**:把 PNG 放进仓库 `incoming/`(git push 或上传页)。
- **引擎侧(本仓库的 Claude)**:`char_intake`/切片入库 → 角色绑骨补间 / 瓦片接 TileMap → 接进游戏。

## 1. 通用产出规格(所有工具都按这个)
- **透明背景 PNG**;不要烘焙地面阴影。
- **侧视、面朝右、全身、单角色**;角色大小一致;**双脚踩同一基线**。
- **一张图一个姿势**;命名 `动作_序号.png`(如 `attack1_2.png`)。
- 帧数:跑 6–8 / 待机 2–4 / 跳·落 各1–2 / 攻击 3–5 / 受击 2–3 / 死亡 4–6。
- **一致性 > 帧数 > 画质**;角色动画**只出关键姿势**,动画由引擎侧绑骨/补间。

---

## 2. 主推:Scenario.gg
把"训练你的风格模型 + ControlNet 控姿势 + 出图"托管成网页/API,一致性最强。

### A. 网页流程(最省心)
1. 注册登录 → **Models → 新建/训练自定义模型**。
2. 上传 **20–50 张同风格参考图**(你找的素材/同一画风)→ 选合适的基础类型 → 开始训练(等几十分钟)。
3. 训练完,用该模型 **Generate**:
   - 写提示词(主体 + 动作),需要锁姿势就开 **ControlNet / Pose(OpenPose)**,喂一张火柴人姿势图。
   - 开**透明背景 / 去背景**选项(没有就出纯色底,引擎侧能扣)。
4. 一个动作一张、按规格出;下载 PNG。
5. 放进 `incoming/`(见第 4 节)。

### B. API 流程(给本地 Claude 自动化)
> 让本地 Claude **先打开 Scenario 官方 API 文档**,按其最新端点实现下面这条链路(参数名以文档为准):
1. **鉴权**:用你的 Scenario **API Key**(放环境变量,别写进代码/仓库)。
2. **训练模型**:上传参考图 → 创建训练任务 → 轮询直到完成 → 记下 `modelId`。
3. **生成**:对该 `modelId` 发起生成请求(prompt + 可选 ControlNet 图 + 透明底选项 + seed)→ 轮询 → 拿结果 URL。
4. **下载**:保存为 `动作_序号.png` 到本地 → 放进仓库 `incoming/` → `git push`。
> 注:具体路径/字段(如 `/v1/models`、`/generate` 之类)**以官方文档为准**,本文不写死,避免过时。

## 3. 备选工具
| 工具 | 怎么"按参考出同风格" | 适合 | 是否有 API/MCP |
|---|---|---|---|
| **Recraft** | 从你的图**创建"风格"**再复用 | 成套一致、透明底、矢量 | 有 API |
| **Leonardo.ai** | Image Guidance(img2img/ControlNet)+ 自训数据集 | 游戏向、风格库多 | 有 API |
| **Midjourney** | **`--sref` 风格参考 + `--cref` 角色参考** | 画质/一致性顶级 | 无官方 API(需自动化网页);无透明底要扣 |
| **PixelLab.ai** | 参考/风格,**直接出走/跑/攻击帧、骨骼** | **像素风 + 要动画** | 有 API/插件 |
| **Ludo.ai** | 参考/风格 | 构思 + 杂项素材 | **有 MCP**(本仓库 `.mcp.json` 已配,本地可用) |
| **Adobe Firefly** | Structure / Style reference | 商用版权安全 | 有 API |

> 像素风优先 PixelLab;高清/手绘风格一致性优先 Scenario/Recraft;要顶级画质且能接受手动抠图用 Midjourney。

---

## 4. 产出怎么交给引擎侧
1. PNG 按 `动作_序号.png` 命名,放进仓库 `incoming/`(上传页 or `git add incoming/ && git push`)。
2. 在本仓库会话里跟 Claude 说:"`incoming/xxx` 是角色/瓦片,做进去"。
3. 引擎侧会:
   - 角色姿势:`python3 tools/char_intake/intake.py incoming/<folder> <name>` → 对齐入库 → 绑骨/补间成动画 → 接控制器。
   - 瓦片集:用 `tools/tilemap/` 自定义上传该 tileset 刷关卡;或我接成 Godot TileMap。
   - 序列帧/图集:切片打包成精灵表 + manifest。

## 5. 省钱省心铁律
1. **先小批试**:先出 **3–5 张关键姿势**跑通"出图→入库→进引擎",确认风格对味,再批量。
2. **锁一致性**:训练模型 / 固定参考图 + 种子 + 同一提示词前缀。
3. **动画走绑骨**:出姿势,不要逐帧硬生成。
4. **密钥安全**:API Key 放环境变量,别提交仓库,用完按需轮换。
