# 出素材工作流（**已弃用**：Scenario 出图+视频 → 选帧后期 → 接入）

> ⚠️ **这条路已验证不可行，不是当前主线**（2026-06 二次验证）：
> "AI 出视频动作 → 选帧拼动画"的死穴是**帧间一致性**——逐帧重画必然漂移，扣图/对齐/去抖的人工成本压过收益。详见 `AI_TOOLS.md` 末尾「实测结论」。
> **当前主线 = 现成包 / 自己画**（见 `ASSET_SOURCING.md`），主角已换成 Mattz Art 免费武士包。
>
> 本文保留作**历史参考**：万一将来出现帧间一致性过关的工具，这套"异色底→选帧→横条→切片"的后期管线仍可复用。下面的横条交付规格对**任何来源**（含现成包切条）都适用。
>
> 旧流程：① **Scenario** 出角色图+视频动作（品红 #FF00FF 底好抠）→ ② **你**用 `tools/sprite-keyer` 选帧/抠图/对齐/导透明横条 → ③ **我** `SpriteSheet.build_from_strips` 切片接入。

## 成品横条规格（交给我前对照）
- **透明背景 PNG**；侧视、**面朝右**（游戏自动镜像朝左）；单角色全身；**双脚同基线**。
- **每帧等宽等高**；横排一行；各帧**水平居中 + 底部对齐**（别帧间漂移）。
- **一个动作一张表**；附一句：动作名、帧宽×帧高、帧数、想要的 fps。
- 帧数参考：跑 6–8 · 待机 2–4 · 跳/落 1–2 · 攻击 3–5 · 受击 2–3。
- **一致性 > 帧数 > 画质**。

## 生成端要点（Scenario）
- **锁一致性**：训练/固定风格 + 固定参考图 + 同种子 + 同提示词前缀。
- **纯色异色底**（亮品红）→ 省后期抠图。
- 视频动作：尽量让**首帧≈尾帧**（循环），后期挑帧好接缝。
- 密钥（用 API 时）放**环境变量**，**别提交仓库**。
