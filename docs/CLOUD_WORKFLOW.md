# 出素材工作流（当前：Scenario 出图+视频 → 你选帧后期 → 我接入）

> **当前主线（2026-06）**：
> 1. **Scenario**：出角色图 + 出**视频动作**（跑/待机/攻击…）。生成时**背景用纯色异色（亮品红 #FF00FF）**，后期抠图最干净。
> 2. **你**：用 `tools/sprite-keyer`（手机网页）**选关键帧 + 抠图 + 对齐 + 导透明横条**。挑帧/抠图/对齐的细节见 `SPRITE_PIPELINE.md`。
> 3. **我（引擎侧）**：横条 → `SpriteSheet.build_from_strips` 切片 → `player.set_sprite_frames` 接进游戏。
>
> 美术是长期事，先**色块占位把玩法做爽**，素材就绪再逐个换（先 idle/run）。

## 成品横条规格（交给我前对照）
- **透明背景 PNG**；侧视、**面朝右**（游戏自动镜像朝左）；单角色全身；**双脚同基线**。
- **每帧等宽等高**；横排一行；各帧**水平居中 + 底部对齐**（别帧间漂移）。
- **一个动作一张表**；附一句：动作名、帧宽×帧高、帧数、想要的 fps。
- 帧数参考：跑 6–8 · 待机 2–4 · 跳/落 1–2 · 攻击 3–5 · 受击 2–3。
- **一致性 > 帧数 > 画质**。

## 生成端要点（Scenario）
- **锁一致性**：训练/固定风格 + 固定参考图 + 同种子 + 同提示词前缀。
- **纯色异色底**（亮品红）→ 省后期抠图。
- 视频动作：尽量让**首帧≈尾帧**（循环），后期挑帧好接缝。
- 密钥（用 API 时）放**环境变量**，**别提交仓库**。
