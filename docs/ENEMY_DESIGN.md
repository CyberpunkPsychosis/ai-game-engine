# 敌人设计 · 弹反玩法专用(同时是找素材/生成的清单)

> 弹反游戏里**敌人就是关卡**。本档定义:① 弹反需要敌人具备什么攻击行为(快慢刀/前摇/危攻击);
> ② 敌人**角色档**(定位+攻击模式+弹反点);③ 每个敌人**需要哪些动画帧**(给你找素材 / 本地 Claude 生成 / 我入库)。
> 主题无关——下面是"角色与行为",套什么皮都行。

---

## 1. 为什么必须"快慢刀 + 清晰前摇"
弹反的乐趣 = **读招 → 卡时机**。要成立,敌人攻击必须满足:
- **前摇(telegraph)**:出手前有一段明显的"举刀/蓄力"姿势 → 玩家靠**看姿势**预判。前摇姿势要**夸张、好认**。
- **快慢混合(快慢刀)**:同一个敌人有**慢刀**(前摇长,好弹)和**快刀**(前摇短,难弹),逼你专注节奏。**这是弹反深度的来源。**
- **节奏可学**:攻击有固定节拍,玩家能背谱、能精通(像音乐)。
- **可读的"种类"**:普通可弹 / 不可弹「危」/ 远程。靠**姿势 + 颜色特效**区分(红光=危)。

> 一句话:**每个攻击 = 一段"前摇姿势(预警)→ 命中帧(弹反点)→ 收招"**,且不同攻击的前摇时长/姿势要能一眼区分。

## 2. 攻击词汇表(每个敌人从中组合)
| 类型 | 表现 | 玩家应对 | 需要的动画 |
|---|---|---|---|
| **慢刀** | 大幅举刀、前摇长 | 从容弹反(教学) | 明显蓄力前摇 + 挥砍 |
| **快刀** | 小幅快出、前摇短 | 反应弹反 | 短促前摇 + 快挥 |
| **连段** | 快快慢 / 三连 | 连续弹反=拼刀 | 多段攻击,各自前摇 |
| **危攻击(红光)** | 不可弹(下砸/突刺) | 闪避/跳/破防 | 夸张蓄力 + 红光特效(引擎加) |
| **假动作 feint** | 举刀停顿骗弹 | 忍住、看真出手 | 前摇中途的"停顿/收回"帧 |
| **远程弹道** | 射箭/投物 | 弹反把它反弹回去 | 瞄准/射击 + 弹道物 sprite |

## 4. 美术/可读性要求(决定你找什么风格的素材)
弹反游戏对美术的**首要要求是"看得懂"**:
- **强剪影**:每个敌人轮廓独特,一眼分清谁是谁。
- **前摇姿势要夸张好认**:举刀/蓄力的那一帧,姿势越大越清楚越好(玩家靠它预判)。
- **快慢刀 = 可区分的起手**:慢刀大幅高举、快刀小幅平刺。
- **颜色编码**:危攻击红光、可弹瞬间高亮——**这些特效引擎加,不依赖素材**;但敌人的"前摇动作"本身必须明显。
- **风格**:高对比、清晰,**像素风天然适合**(剪影干净)。

## 5. 每个敌人的动画清单(找素材/生成/入库用)
> 一个能打的弹反敌人,**最少**要这些(帧数参考 `CHARACTER_POSES.md`):
- `idle`(2–4)、`walk/approach`(4–6)
- **每种攻击各一组**:`windup 前摇`(2–3,**最重要**)+ `attack 挥出`(2–4)
- `perilous 危攻击`(若有):蓄力(3)+ 砸/刺(2–3)
- `hurt 受击`(2)、`death 死亡`(4–6)
- 可选:`parried 被弹踉跄`(2)、`block 格挡`(盾卫用)
- **Boss 额外**:多套攻击段 + 阶段切换 + 处决被杀演出。

## 6. 给你 / 本地 Claude 出素材的提示要点
- 让 AI **每个攻击单独出"前摇姿势"和"挥出姿势"**(一姿势一张,侧视朝右,透明底,统一画幅,脚底对齐)。
- **强调"夸张的蓄力姿势"**(prompt 里写 big wind-up / telegraphed pose),否则弹反读不出来。
- 保持**同一画风**(训风格模型 / 风格参考,见 `CLOUD_WORKFLOW.md`)。
- 出完丢 `incoming/` → 我 `char_intake` 入库 + 接成敌人 AI(前摇→命中帧→可弹判定)。

## 7. 引擎侧我来做的(不依赖素材也能先灰盒验证)
