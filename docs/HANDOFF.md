# 交接 · HANDOFF（新会话从这里接手）

> 写给**新开的 Claude 会话**：用户换会话是因为某些会话不能联网。这份文档让你不用回看聊天
> 就能接着干。先读完本文，再看 `CLAUDE.md` 和 `README.md`。

## 1. 我们在做的事
做一个**网页版「怪物导演」画布工具**，解决一个具体痛点：**用户嘴上描述不清怪物的行动路线**
（"三只螳螂先不动，一只冲过来戳几下退回，另两只从空中俯冲…"）。于是给他一个画布，让他
**把路线画出来**，再把结果发给你来实现。

**这个工具不是用来设计整个机制/数值的**，只补"行动路线/走位/弹道"这一块。用户原话：
> "这个画布弥补了我描述不清的怪的行动路线的问题，并不是为了解决所有机制的。"

## 2. 工作流（很重要，按这个节奏来）
**分阶段、一段一段做，不要想一次做完整只怪/整个 Boss：**
1. 用户在工具里**画一个阶段**的行动（如开场怎么动），点「复制给AI」得到一段文本，发给你。
2. 你在 Godot 里**实现这一段**，怪要带 `tunables()`（实时调参项）。
3. 用户**试玩**，用游戏内 `⚙调参` 面板（`scripts/autoload/dev_tools.gd`，电脑 F1 / 手机左上角按钮）
   **拖滑块即时调**速度/距离/伤害等手感，不满意发数值给你或自己调好。
4. 满意后用户**画下一阶段**，重复。
- **数值/手感由你定**（给合理默认），用户靠试玩微调。位置是"大概位置"，最终还会跟用户聊确认。

## 3. 工具：`tools/enemy-designer/index.html`
纯前端单文件（vanilla JS + canvas），无依赖。本地 `localStorage` 自动存。CI 部署到
`<pages>/tools/enemy-designer/`。**改完用 `node --check` 验证内联脚本**（见下方"如何验证"）。

### 用户能在画布上做什么
- **摆位**：放怪 / 放玩家到 2D 任意位置（**点在空中 = 该怪标记"会飞"**，地面线 y=GY）。
- **画路线**：选中一只怪 → 在画布依次点出它走/飞的路线点（折线）。每个**路线点**可设动作：
  路过 / 到此攻击N下 / 连击 / **发射弹道** / 召唤 / 说话 / 停顿（+ 停顿秒数）。
- **画弹道（远程怪）**：把某路线点动作设为「到此发射弹道」→「✏画此弹道」→ 在画布点出弹丸轨迹。
  **「弧线」开关**用 Catmull-Rom 平滑（中间点抬高 = 抛物线/吊射）；可设弹速、发数（多发 0.25s 错峰）。
- **每只怪**：会飞、速度、**出场延迟**（用延迟实现"先 B 冲出，2 秒后 A/C 俯冲"的错峰编排）。
- **▶播放**：沿路线动画演示走位/攻击闪光/召唤/台词/弹丸飞行，让用户看节奏对不对。
- **复制给AI**：导出**带坐标的可读行动路线 + JSON**。内置「三螳螂+弓手」示例。

### 你会收到的文本长这样（坐标系：画布 760×420，地面 y≈374，y 越小越高/空中）
```
场景: 三螳螂+弓手（侧视，画布 760×420，地面 y=374，y越小越高/空中）
玩家起点: (200,374)
怪与路线:
  - 螳B 起点(600,374) 速度160
      路线: 起点 → (300,374)[连击3下] → (600,374)[路过·停0.4s]
  - 弓手 起点(710,374) 速度160 延迟1s出场
      路线: 起点 → (710,374)[发射弹道(弧线) 3发 弹速240 轨迹:此点→(480,110)→(215,356)·停0.2s]
```
把这些坐标按比例映射到你实际的游戏房间尺寸即可（工具坐标只表达相对布局/高低/弧度）。

## 4. 怎么把它实现成游戏里的怪
- 敌人基类：`scripts/enemy.gd`（`class_name Enemy`，继承 `Actor2D`）。每招是一个 `move` 字典
  （anim/reach/size/命中帧/dmg/perilous/recover/weight/range/backoff），AI 单位同时只 1 个出手。
- **务必实现 `tunables() -> Array`**（基类已有通用项：出手距离/保持距离/移速/索敌等），
  子类追加这只怪特有的可调项，这样 DevTools 面板能实时拖。把"可能要调"的数值都暴露成 `@export` + tunables。
- 受击/特效：`scripts/components/`（health/hitbox/hurtbox）、`scripts/autoload/`（FX 闪白/Juice 顿帧震屏）。
- 弹道/远程：实现一个沿点列（可贝塞尔/抛物线）飞行的 projectile，参数对应工具里的轨迹点/弹速/发数。
- 敌人加入 `enemy` group（DevTools 靠 group 找当前怪并铺调参/透视）。

## 5. 如何验证（无需本地 Godot）
- 工具改动：抽出 `<script>` 跑 `node --check`：
  ```bash
  python3 -c "import re;s=open('tools/enemy-designer/index.html').read();open('/tmp/c.js','w').write('\n'.join(re.findall(r'<script>(.*?)</script>',s,re.S)))"
  node --check /tmp/c.js
  ```
- 游戏改动：CI（`.github/workflows/build.yml`）导出网页版到 GitHub Pages + Windows exe 工件。
