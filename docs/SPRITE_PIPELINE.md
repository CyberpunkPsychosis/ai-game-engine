> 分工（2026-06 定）：**你**负责「生成视频 → 挑关键帧 → 抠图 → 对齐 → 导出成品 sheet」（因为你要微调）；
> **我（引擎侧 Claude）**只负责「切帧成 `SpriteFrames` + 接进游戏」。
> 已跑通的实例：主角 8 帧跑步 = `art/timestop/hero/run_strip.png`（见 `game.gd` 的 `_load_hero_sprites()`）。
> 配套：成品规格见 `CLOUD_WORKFLOW.md` 第 1 节；学画像素见 `LEARN_GODOT.md` 旁的学习链接。

## 0. 你交给我的「成品 sheet」长啥样（交付前逐条对照）
- **透明背景 PNG**（底抠干净，别留地面阴影/光晕）。
- **侧视、面朝右**（游戏里朝左会自动水平镜像，你只做朝右那套）。
- **单角色、全身、双脚踩同一基线**。
- **每帧等宽等高**（uniform cell）：横排一行，或规整网格都行。
- **各帧水平居中 + 底部对齐（脚对脚）**，别让角色在格子里帧间漂移 → 否则游戏里会抖/滑。
- **一个动作一张表**：`run.png` / `idle.png` / `attack.png` …（别把多动作混一张）。
- 发我时**附一句**：动作名、帧宽×帧高、帧数（几行几列）、想要的 fps。
- 帧数参考：跑 6–8 · 待机 2–4 · 跳/落 各 1–2 · 攻击 3–5 · 受击 2–3 · 死亡 4–6。
- **一致性 > 帧数 > 画质**。

## 1. 怎么从视频里「挑关键帧」（核心，先学这个）

**关键帧（key pose）= 动作里"姿势走到极值/转折"的那几帧，不是均匀截图。** 抓住关键姿势，动作就立住了。

### 走/跑循环 —— 背下这 4 个关键姿势
一条腿走一轮 = 这 4 帧；两条腿一个完整循环 = 8 帧（你那张跑步就是 8 帧）：

1. **接触 Contact**：两腿**前后张到最开**（一脚跟、一脚尖着地）。← 最好认
2. **下沉 Down**：腿并拢吃重，**身体最低点**（看头顶最低）。
3. **通过 Passing**：后腿抬起**从支撑腿旁边交错经过**，腿基本竖直并拢。
4. **抬起 Up**：蹬腿伸展，**身体最高点**（看头顶最高）。

**在视频时间轴上怎么找**：
- 拖时间轴，**腿张到最大**的那帧 = 接触帧（每条腿各一个，循环里有两个）。
- **腿交叉/并拢**的那帧 = 通过帧。
- 盯**头顶高度**：最低 = 下沉、最高 = 抬起。

### 怎么保证「无缝循环」
- **最稳的做法：生成时就让首帧=尾帧**（Seedance 的 `lastFrameImage` / 循环选项；见 CLAUDE.md 的 AI 动画流程）。这样天然闭环，随便等间隔抽帧都接得上。
- 没有循环选项时：**先按时间等分抽 N 张**做底，**再逐张微调到最近的关键姿势**（等间隔 + 看姿势双保险）。
- 抽完**把第 1 帧和最后 1 帧叠一起（洋葱皮）**看接不接得上；接不上就挪一帧。

### 其它动作怎么挑
- **idle 待机**：2–4 帧，呼吸（胸口/头上下 1–2px），**首≈尾**。
- **攻击**：3–5 帧 = **蓄力(预备) → 挥出(命中) → 收招**。⚠️**一定要有蓄力帧**，否则动作像瞬移。
- **跳/落**：起跳 1 帧 + 顶点/下落 1 帧即可。
- 播放帧率：idle ~6fps · 走 ~8 · 跑 ~10–12 · 攻击 ~10–14（fps 我这边可随时改）。

## 2. 抠图（背景移除）用什么工具

### 🔑 最重要的前置技巧（决定抠图难易）
**生成时就让背景是「纯色 + 角色身上没有的颜色」**（亮品红 #FF00FF 或亮绿 #00FF00 做 chroma key）。
→ 抠图秒变干净。**千万别用接近肤色/白色/米黄的背景**（你那张米黄底我抠得动，但边缘会有点残留；换成亮品红会干净得多）。

### 工具
| 工具 | 免费 | 怎么抠纯色底 |
|---|---|---|
| **Aseprite** ⭐ | 付费 | 选区 → `Select > Same Color`(魔棒) 点背景 → 删；或 `Replace Color`。**还能顺手对齐+导表，一站式** |
| **Photopea**（网页版 PS） | 免费 | 魔棒 / `选择 > 色彩范围` → 删 |
| **GIMP** | 免费 | 按颜色选择 / 模糊选择(fuzzy) → 删 |
| **Photoshop** | 付费 | `选择 > 色彩范围` / 魔棒 |

### ⚠️ 保留角色内部的白（袜子/鞋/眼睛）
- 用「**从边缘点背景、选连通区域**」（魔棒点背景 + 勾 **contiguous/连续**）或**洪水填充**，只删**和边缘连通**的背景。
- **别用全局颜色阈值、也别用 AI 抠图(rembg/isnet)** —— 那会把角色内部的白也吃掉、还会**帧间闪烁**。**我们之前就栽在这（袜子一闪一闪/变黑）。**
- 抠完放大看边缘有没有**残留底色光晕(halo)**，有就「收缩选区 1px 再删」或去边。

## 3. 对齐（帧间不抖的关键）

**目标**：每帧角色**脚踩同一条基线、水平锚点一致**。否则进游戏会抖/滑。

- **Aseprite 做法**（推荐）：把每帧放进**时间轴**，开**洋葱皮(onion skin)**（半透明叠显前后帧），逐帧把角色挪到对齐；**固定画布大小**，加一条**水平基线参考图层**当地面，每帧脚踩上去。
- **别每帧紧裁**（紧裁会漂移）；要**统一画框 + 底部对齐**。
- 导出：Aseprite 直接 **Export Sprite Sheet**（横排、固定帧尺寸、透明底）。
- 朝向：源视频若**面朝左**，整张**水平翻转**成朝右再交我。

## 4. 导出 & 交给我（引擎侧做啥）
1. 一个动作一张 PNG（横排或规整网格，等格、透明底）。
2. 放进仓库（上传页 / `git push`）或直接发我，**说清**：动作名、帧宽×帧高、帧数、fps。
3. 我：`scripts/sprite_sheet.gd` 的 `build_from_strips`（横排）/ `build`（网格）切片 → `player.set_sprite_frames(sf, 帧高, 脚y)` 进游戏；多动作就多张表分别接（接一个就替掉一个占位帧）。
4. 当前占位状态：只有 `run` 是真动画；`idle/jump/fall/attack/dash` 还是单帧占位 → **先补 `idle` 收益最大**（站着就不尴尬了）。

## 6. 一句话流程
**生成时用纯色异色底**（亮品红）→ **首帧=尾帧**求无缝 → **等间隔抽帧再对到关键姿势** → **连通区抠底(留内部白)** → **Aseprite 洋葱皮对齐脚基线** → **导横排表** → 发我切帧接入。
