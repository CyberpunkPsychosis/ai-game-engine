# 瓦片 & 装饰素材清单 —— 按「凝界」世界观（刹那 TimeStop）

> 角色精灵走现成素材/人工；**地形瓦片 + 装饰让 AI 出**（规整、可平铺、不挑帧间一致性）。
> 模型：`model_retrodiffusion-tile`。(颜色暂不锁定,2026-06 定;配色随素材来。)

## 出图约定
- 色板:**暂不锁定**(颜色未定,按工具默认出;以后定了色调再统一量化)。
- **RD Tile 的 `style` 模式 ↔ 用途**：
  | 模式 | 干啥 | 给谁用 |
  |---|---|---|
  | `tileset` | 整套自动拼接地形(边/角/中/过渡) | 地面、平台、岩体 |
  | `single_tile` | 单张无缝可平铺 | 背景墙、地板填充 |
  | `tile_object` | 瓦片对齐的物件 | 灯笼/齿轮/晶簇/门/长椅/尖刺 |
  | `scene_object` | 较大场景物 | 鸟居/巨钟/远景剪影/Boss 背景 |
  | `tileset_advanced` / `tile_variation` | 进阶/同区变体 | 防单调用 |

---

## A. 全局通用（所有区共用）
- **A1 主地形 auto-tile**（地面/平台顶，可踩）— `tileset` ★最先做
- **A2 实心岩体/墙内填充**（地形内部）— `tileset`
- **A3 悬空平台**（窄条，带边缘冷高亮，可站立）— `tile_object` / `tileset`
- **A4 背景近景墙**（不可踩，压一档暗）— `single_tile`
- **A5 远景视差层**（剪影建筑/山，冷蓝，分层）— `scene_object`
- **A6 凝界悬停粒子贴图**（雨丝/落叶/火星，冷白小点；现为代码画，可换贴图）— `tile_object`
- A7 暗角 / 扫描线 / 冷调 → **已有后处理 shader**，不用瓦片

## B. 玩法功能 tile（接机制）
- **B1 门 / 出口**（发光门框，区域色描边）— `tile_object`（接 `exits`）
- **B2 存档长椅**（暖色；现为代码占位）— `tile_object`（接 `benches`）
- **B3 尖刺 / 陷阱 hazard** — `tile_object`
- **B4 断坑边缘 / 悬崖收口**（地形左右收边）— `tileset` 边角块（接 `solids` 的坑）
- **B5 被冻物=立足点** 的亮蓝描边 → **VFX 代码**，不用瓦片

## C. 按生态区（每区换主色 + 专属装饰）

### C1 神社街（**青蓝主**）— 第一个完整区
- 地形：青灰石板 / 苔石台阶 — `tileset`
- 背景：町屋木墙 / 格子窗 / 瓦顶 — `single_tile` + `scene_object`
- 装饰物件：鸟居、石灯笼（笼内**冻火=亮蓝**）、注连绳/纸垂、赛钱箱、狛犬石像、绘马墙、石阶栏杆 — `tile_object` / `scene_object`
- 悬停点睛：**落樱花瓣**（半空停，冷白粉）— `tile_object`
- 暖点缀（少量）：纸灯笼（暖橙）

### C2 钟楼内部（**琥珀 / 铜主**）
- 地形：黄铜地板 / 齿轮嵌石 / 铜管平台 — `tileset`
- 背景：发条墙 / 大钟面 / 管道 — `single_tile` + `scene_object`
- 装饰物件：巨齿轮、钟摆、发条弹簧、铜管阀门、铜灯、**卡住的指针** — `tile_object` / `scene_object`
- 悬停点睛：凝固的火星 / 铜屑（暖金）— `tile_object`

### C3 深渊（**紫红主**）
- 地形：紫黑岩 / 晶体地台 / 裂隙 — `tileset`
- 背景：深渊岩壁 / 血色根须 / 残破符文 — `single_tile` + `scene_object`
- 装饰物件：紫晶簇、血色根须、悬浮符文牌、骨堆 — `tile_object`
- 破调色板冲击：**血月**（红）— `scene_object`
- 悬停点睛：紫红灰烬 / 血滴（半空停）— `tile_object`

### C4 Boss / 钟枢（**专属色，故意破调色板**）
- 地形：钟枢核心地台（黑金 + 亮蓝裂纹）— `tileset`
- 场景：**巨钟**、时之碎片群、凝固灾厄底座、悬龙 Boss 背景（火龙带橙）— `scene_object`

## D. 优先级（先做能玩的）
1. **A1 主地形 + A3 平台 + B1 门 + B4 坑边** → 房间从色块 `solids` 升级成真瓦片（先把现有 3 房刷出来）
2. **C1 神社街整套**（第一个完整生态区）
3. A4/A5 背景 + 视差层
4. C2 钟楼 / C3 深渊
5. C4 Boss/钟枢

## E. 接入流程（其中不花 CU 的先做）
1. RD Tile 出整套 → 切成 Godot **TileSet** 资源（含碰撞形状）
2. 房间 JSON 的 `solids` 升级成 **TileMap 坐标层**（或保留 solids 做碰撞、TileMap 只做视觉）
3. `game.gd` 渲染：地形从色块 `_draw` 改成 `TileMap`；装饰作为 `instances`（对接 composer 编辑器的 `instances`）
4. 装饰/门/长椅 → 接现有 `exits`/`benches`/`markers`

> 省 CU 做法：**先把 A1/A3/B1/B4 这几套地形瓦片出齐 + 打通 TileMap 接入管线**，验证"AI 瓦片→游戏关卡"这条路；之后再按区批量出装饰。
