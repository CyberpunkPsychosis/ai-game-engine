# 12 周学习路线图

> 原则:**每天 1–2 小时,稳过猛。** 重点不是"学完",是"每周都做出一个能玩/能看的小东西"。
> 节奏假设你业余时间学;有更多时间就压缩周数。

## 阶段一(第 1–3 周):先尝甜头,建立手感
**主线:选一个引擎,跑通"从开工程到一个能动的角色"。**
- [ ] 装好引擎 + 编辑器(Godot 直接下;Unity 见 `unity/setup-unity-mcp.md`)
- [ ] 跟 1 个完整入门教程做出"角色能左右走 + 跳"(Godot 或 Code Monkey 的 Unity 课)
- [ ] **画画**:每天 15 分钟像素画,目标——画出一个 16×16 的小角色站立图
- [ ] 周末小目标:让你画的小人,在引擎里能走能跳 ✅(打通"美术→引擎"全链路)

## 阶段二(第 4–6 周):做一个"超小"完整游戏
**别贪大。** 一个屏幕、一个机制、能开始能结束,就算完整。
- [ ] 加敌人 / 障碍 + 碰撞 + 死亡/重开
- [ ] 加一个计分或一个目标(吃到星星/到达终点)
- [ ] 加最基础的音效 + 一个标题画面
- [ ] **画画**:画一组动作帧(站/走/跳),理解"逐帧动画"
- [ ] 周末小目标:把这个超小游戏**给一个朋友玩**,看他卡在哪

## 阶段三(第 7–9 周):上手 AI 出图 + 风格一致性
**这是你项目真正的瓶颈,单独花时间攻。**
- [ ] 按 `art/learning-path.md` 跑通一次 AI 出图工作流(文生图→图生图迭代)
- [ ] 试一次 LoRA / 风格锁定,让同一角色多张图风格一致
- [ ] 定一份自己的 **style guide**(配色/线条/细节程度),以后所有素材对齐它
- [ ] **引擎**:把 AI 出的素材切成 spritesheet 导进引擎用起来

## 阶段四(第 10–12 周):AI 驱动 + 独立能力合体
**目标达成:你能独立做小游戏 + 会用 AI 加速。**
- [ ] (可选)按 `unity/setup-unity-mcp.md` 搭通 Unity MCP,体验"AI 直接操作编辑器"
- [ ] 在你的 Godot 主项目里独立做一只完整的怪(用你之前的"怪物导演"工具画路线)
- [ ] 复盘:哪些环节你已经能独立做?哪些还想让 AI 帮?——这就是你未来的分工

---

## 防放弃小贴士
- **进度要可见**:每周存档/截图,回头看会很有成就感。
- **卡住先问搜索/社区,再问 AI**:自己解出来一次,比 AI 给十次答案记得牢。
- **别比较**:你是 AI 驱动 + 自学双线,节奏本来就和科班不一样,对自己耐心点。
