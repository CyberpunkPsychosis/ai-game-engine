# 学习中心 · Learning Hub

> 目标:**让自己不靠 AI 也能做游戏。** 自己懂了引擎、懂了画画,以后再用 AI 驱动开发,
> 才是如虎添翼,而不是离了 AI 就瘫痪。

这是一个**独立的学习分支**(`claude/learning-hub-unity-godot-art`),不影响你现有的
Godot 游戏主项目(在 `claude/game-engine-claude-chat-mbKeN` 分支上)。这里只放**学习资料、
课程规划、环境搭建指南**,你可以放心边学边改。

## 你现在的定位(写给自己看)
- 不会引擎开发 → 要学**操作**(不是从零造引擎,而是会用)
- 不会画画 → 要学**够用的美术**(像素画 + AI 出图,不是当大师)
- 想要的最终状态 → **不和 AI 对话也能独立做出游戏**,AI 只是加速器

## 三条学习线
| 线 | 文件夹 | 一句话 |
|---|---|---|
| 🟢 Godot | [`godot/`](./godot/learning-path.md) | 你已有项目,拿它当练手沙盒,最快见效 |
| 🔵 Unity | [`unity/`](./unity/learning-path.md) | 业界主流,学了好找资料/教程多;含 Unity MCP 搭建 |
| 🎨 画画 | [`art/`](./art/learning-path.md) | 像素画(零基础友好) + AI 出图(解决一致性) |

## 怎么用这个学习中心
1. 先读 [`00-roadmap.md`](./00-roadmap.md) —— 12 周的阶段计划,别一口吃成胖子。
2. 三条线**不要同时猛攻**。建议:**主攻 1 个引擎 + 每天一点画画**,另一个引擎当扩展。
3. 每学完一小块,就在你的 Godot 主项目里**真的做一个小东西**出来(哪怕一个会动的怪)。
   —— 做出来 > 看完教程。手比眼睛记得牢。

## 引擎到底选哪个?(给你的诚实建议)
- **想最快做出东西、且已有 Godot 项目** → **主攻 Godot**,Unity 当了解。
- **想学就业/资料最多/将来用 Unity MCP 让 AI 直接操作编辑器** → **主攻 Unity**。
- **两个都想会** → 先把**一个**学到能独立做出小游戏,再碰第二个。同时学两个最容易放弃。

> 详细取舍见 [`unity/learning-path.md`](./unity/learning-path.md) 开头的对比表。
